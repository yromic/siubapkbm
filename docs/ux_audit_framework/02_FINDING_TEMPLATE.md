# Enterprise UX Finding Template

All heuristic audit findings generated by auditors MUST strictly adhere to this template structure. This ensures consistency, technical clarity, and actionable guidance for engineering teams.

---

```markdown
### [FINDING-ID]: [Short, Descriptive Title of Usability Issue]

#### 1. Location & Context
- **Module**: [e.g., Student Management / Academic Grading / Authentication]
- **Screen / View**: [e.g., Student Enrollment Modal / Grade Roster Table]
- **URL Path**: [e.g., /portal/students/enroll]
- **Affected Persona**: [e.g., Administrator / Teacher / Parent]
- **User Goal**: [Brief description of what the user is attempting to accomplish]

#### 2. Heuristic Classification
- **Primary Heuristic Violated**: [e.g., Heuristic #1: Visibility of System Status]
- **Secondary Heuristic(s)**: [e.g., Heuristic #5: Error Prevention]
- **Severity Level**: [Critical / High / Medium / Low / Cosmetic]

#### 3. Empirical Evidence
- **Visual Capture**: ![Description](file:///path/to/evidence_screenshot.png)
- **DOM / Code Snippet**:
  ```tsx
  // Exact rendered snippet or interaction code demonstrating the flaw
  ```
- **Observed Behavior**: [Detailed explanation of what currently happens when the user interacts with the system]

#### 4. Impact Analysis
- **User Impact**: [How this affects the user's task completion, time, and mental model]
- **Business / Operational Impact**: [How this impacts data integrity, support volume, or operational efficiency]

#### 5. Expected Behavior
[Clear description of how the system should behave to fulfill usability standards]

#### 6. Actionable Recommendations
- **Quick Win (Immediate Fix)**:
  [Simple UI or text modification that can be deployed quickly]
- **Long-Term Strategic Fix**:
  [Full design system or architectural solution for permanent improvement]

#### 7. Technical & Regression Considerations
- **Implementation Effort**: [Low / Medium / High]
- **Regression Risk**: [Low / Medium / High]
- **Dependencies**: [e.g., Requires state management update, API error message overhaul, etc.]
```
